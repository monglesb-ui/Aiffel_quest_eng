# 에르메스 에이전트 기반 머신러닝 실험 제어 프레임워크

## 설명 가능한 반복 학습 실험 자동화를 중심으로

### 초록

본 연구는 머신러닝 학습 과정을 단일 모델 최적화 문제가 아니라, 반복적인 실험 제어 과정으로 재정의한다. 일반적인 머신러닝 실험에서는 연구자나 개발자가 학습 결과를 확인한 뒤 learning rate, epoch, optimizer, scheduler 등의 설정을 수동으로 조정한다. 이 과정은 단순한 하이퍼파라미터 탐색을 넘어, 학습 상태를 해석하고 다음 실험 방향을 결정하는 반복적 의사결정 과정이다.

본 논문은 이러한 실험 흐름에 에르메스 에이전트를 연결하는 프레임워크를 제안한다. 에르메스 에이전트는 학습 로그와 검증 지표를 관찰하고, 과적합, 과소적합, 수렴 정체, 학습 불안정성과 같은 상태를 해석하며, 사전에 정의된 권한 범위 안에서 다음 실험 설정을 결정한다. 제안 구조의 핵심은 성능 최적화만이 아니라, 실험 결정 과정의 설명 가능성, 재현성, 기록 가능성을 함께 확보하는 데 있다.

본 연구는 기존 AutoML 및 하이퍼파라미터 최적화 접근과 달리, 최적 조합 탐색보다 실험 흐름의 제어와 결정 근거의 기록에 초점을 둔다. 실험 결과는 향후 실제 학습 환경에서 에르메스 에이전트 적용 전후의 성능, 실험 횟수, 수렴 안정성, 사람 개입 횟수, 결정 로그의 설명 가능성을 비교하여 검증한다.

**주요어:** 에르메스 에이전트, 머신러닝 실험 자동화, 실험 제어, AutoML, 하이퍼파라미터 조정, 설명 가능한 자동화

## 1. 서론

머신러닝 모델 학습은 반복적인 실험 과정이다. 연구자는 데이터를 준비하고 모델을 학습시킨 뒤, 학습 결과를 관찰하고, 성능이 충분하지 않거나 학습이 불안정할 경우 학습 파라미터나 실험 조건을 수정한다. 이 과정은 일반적으로 사람이 직접 수행하며, 학습 로그와 검증 성능을 해석하는 판단 능력에 크게 의존한다.

기존의 머신러닝 자동화 연구는 주로 AutoML, 하이퍼파라미터 최적화, 모델 구조 탐색, 파이프라인 자동화에 초점을 두었다. 이러한 접근은 성능 향상과 탐색 효율성 측면에서 유용하지만, 실제 실험 과정에서 발생하는 판단의 맥락을 충분히 드러내지 못하는 경우가 있다. 예를 들어 어떤 실험을 왜 중단했는지, 왜 learning rate를 낮췄는지, 왜 epoch를 연장했는지와 같은 결정 과정은 실험 결과만으로는 명확히 설명되기 어렵다.

본 연구는 머신러닝 학습을 단순한 최적화 절차가 아니라, 에이전트가 반복적으로 관찰, 해석, 결정, 실행, 기록을 수행하는 실험 제어 과정으로 바라본다. 이를 위해 본 논문은 에르메스 에이전트를 ML 학습 실험 흐름에 연결하는 프레임워크를 제안한다.

본 연구의 핵심 주장은 다음과 같다.

> 머신러닝 학습 자동화는 최적 하이퍼파라미터를 찾는 문제에만 머물지 않고, 반복 실험의 의사결정 과정을 설명 가능하게 제어하는 문제로 확장될 수 있다.

본 논문의 주요 기여는 다음과 같다.

1. 머신러닝 학습 과정을 에이전트 기반 반복 실험 제어 문제로 재정의한다.
2. 에르메스 에이전트의 관찰, 해석, 결정, 실행, 기록 구조를 ML 실험 루프에 연결한다.
3. 학습 파라미터와 실험 흐름을 조정하는 에이전트 실행 구조를 제안한다.
4. 기존 AutoML과 달리 실험 결정 과정의 설명 가능성과 재현성을 평가 기준에 포함한다.
5. 향후 실험 결과를 통해 에이전트 기반 실험 제어의 가능성과 한계를 검증할 수 있는 평가 틀을 제시한다.

## 2. 관련 연구

### 2.1 AutoML

AutoML은 데이터 전처리, 모델 선택, 하이퍼파라미터 탐색, 모델 평가 등의 과정을 자동화하는 연구 분야이다. AutoML은 사용자의 개입을 줄이고 더 나은 모델 성능을 얻는 데 목적이 있다. 그러나 많은 AutoML 접근은 최종 성능이나 최적 조합 탐색에 초점을 두며, 각 실험 결정이 어떤 관찰과 해석을 통해 이루어졌는지를 충분히 설명하지 않는 경우가 있다.

본 연구는 AutoML의 목표를 대체하려는 것이 아니라, AutoML이 상대적으로 덜 강조해온 실험 의사결정의 설명 가능성과 실험 흐름 제어를 보완하려는 접근이다.

### 2.2 하이퍼파라미터 최적화

하이퍼파라미터 최적화는 grid search, random search, Bayesian optimization, evolutionary search 등의 방식으로 모델 성능에 영향을 미치는 설정값을 탐색한다. 이러한 방법들은 탐색 공간과 목적 함수가 명확할 때 효과적이다.

그러나 실제 실험에서는 단순히 탐색 공간을 넓게 설정하는 것만으로 충분하지 않을 수 있다. 학습 곡선의 형태, 과적합 여부, 수렴 정체, 학습 불안정성 등은 다음 실험의 방향을 정하는 중요한 신호가 된다. 본 연구는 이러한 신호를 에르메스 에이전트가 해석하고, 제한된 행동 공간 안에서 다음 실험을 결정하는 구조를 제안한다.

### 2.3 실험 추적과 MLOps

MLflow, Weights & Biases와 같은 도구는 실험 설정, metric, artifact, 모델 버전 등을 기록하고 비교할 수 있도록 지원한다. 이러한 도구는 실험 관리와 재현성 확보에 중요한 역할을 한다.

다만 실험 추적 도구는 주로 기록과 시각화에 초점을 두며, 다음 실험을 어떤 기준으로 설계할지는 대체로 사용자에게 맡긴다. 본 연구의 에르메스 에이전트는 이러한 실험 로그를 단순히 저장하는 것이 아니라, 다음 실험 결정을 위한 입력으로 사용한다.

### 2.4 에이전트 기반 워크플로

에이전트 기반 워크플로는 목표를 가진 실행 주체가 환경을 관찰하고, 상태를 해석하며, 적절한 행동을 선택하고, 그 결과를 바탕으로 다음 행동을 반복하는 구조를 갖는다. 본 연구는 이러한 에이전트적 실행 구조를 머신러닝 학습 실험에 적용한다.

본 논문에서 에르메스 에이전트는 모델 자체가 아니라, 모델 학습 실험을 제어하는 상위 실행 주체이다. 따라서 모델은 데이터를 학습하고 metric을 산출하며, 에르메스 에이전트는 그 metric을 바탕으로 실험 흐름을 조정한다.

## 3. 에르메스 에이전트의 정의

본 연구에서 에르메스 에이전트는 머신러닝 학습 실험을 반복적으로 관찰하고 조정하는 실행 주체로 정의된다. 에르메스 에이전트는 모델의 내부 파라미터를 직접 학습하는 것이 아니라, 학습을 둘러싼 실험 조건과 흐름을 제어한다.

에르메스 에이전트의 기본 순환 구조는 다음과 같다.

```text
관찰 Observation
→ 해석 Interpretation
→ 결정 Decision
→ 실행 Execution
→ 기록 Logging
→ 반복 Iteration
```

각 단계의 역할은 다음과 같다.

| 단계 | 역할 | 예시 |
|---|---|---|
| 관찰 | 학습 결과와 실험 상태를 수집한다 | train loss, validation loss, accuracy, epoch별 metric |
| 해석 | 현재 학습 상태를 진단한다 | 과적합, 과소적합, 수렴 정체, 학습 불안정성 |
| 결정 | 다음 실험 행동을 선택한다 | learning rate 조정, epoch 연장, early stopping |
| 실행 | 선택한 설정으로 실험을 수행한다 | 새 run 생성, checkpoint 기반 재학습 |
| 기록 | 판단 근거와 결과를 저장한다 | 변경 이유, 변경값, 결과 metric |
| 반복 | 개선 여부에 따라 다음 실험을 이어간다 | 계속, 중단, 후보 전략 변경 |

에르메스 에이전트는 완전 자율 시스템이 아니라, 사전에 정의된 권한과 제약 안에서 작동하는 실험 제어 에이전트이다. 이 제한은 재현성, 안정성, 데이터 누수 방지, 실험 로그의 신뢰성을 위해 필요하다.

## 4. 제안 프레임워크

제안 프레임워크는 ML 학습 모듈과 에르메스 에이전트 모듈을 분리한다. ML 모델은 데이터를 학습하고 결과를 산출하며, 에르메스 에이전트는 해당 결과를 관찰하여 다음 실험 조건을 결정한다.

```text
Dataset
→ Training Module
→ Metric Observer
→ Hermes Agent Controller
→ Experiment Decision
→ Updated Training Configuration
→ Re-training
```

### 4.1 Training Module

Training Module은 실제 모델 학습을 수행한다. 이 모듈은 데이터셋, 모델 구조, optimizer, loss function, epoch, batch size 등의 설정을 입력으로 받아 학습을 수행하고, 학습 결과를 metric 형태로 반환한다.

생성되는 정보는 다음과 같다.

- train loss
- validation loss
- validation accuracy 또는 task-specific metric
- epoch별 learning curve
- checkpoint
- 학습 시간
- 에러 또는 중단 상태

### 4.2 Metric Observer

Metric Observer는 Training Module에서 생성된 학습 결과를 수집하고, 에르메스 에이전트가 해석할 수 있는 형태로 정리한다. 단순한 최종 성능뿐 아니라 시간에 따른 변화가 중요하다.

관찰 항목은 다음과 같다.

- validation loss의 개선 여부
- train loss와 validation loss의 차이
- metric plateau 발생 여부
- loss curve의 진동 여부
- best checkpoint 발생 시점
- early stopping 조건 충족 여부

### 4.3 Hermes Agent Controller

Hermes Agent Controller는 본 프레임워크의 핵심 모듈이다. 이 모듈은 Metric Observer가 제공한 정보를 바탕으로 현재 실험 상태를 해석하고, 다음 실험 행동을 결정한다.

에이전트는 다음과 같은 질문을 수행한다.

- 현재 모델은 과적합 상태인가?
- 현재 모델은 과소적합 상태인가?
- 학습이 정체되었는가?
- learning rate가 너무 크거나 작은가?
- 추가 학습이 의미 있는가?
- 실험을 중단하고 다른 전략으로 이동해야 하는가?

### 4.4 Rule and Constraint Manager

Rule and Constraint Manager는 에르메스 에이전트의 행동 범위를 제한한다. 이는 에이전트가 임의로 실험 기준을 바꾸거나, test set을 이용해 의사결정을 하거나, 이전 실험 기록을 훼손하는 것을 방지한다.

제약 조건의 예시는 다음과 같다.

- test set은 최종 평가에만 사용한다.
- validation metric은 하나의 실험 세션 안에서 임의로 변경하지 않는다.
- 모든 실험 설정과 결과는 로그에 남긴다.
- 허용된 후보 범위를 벗어난 하이퍼파라미터 변경을 금지한다.
- 실패한 실험도 기록한다.

### 4.5 Experiment Logger

Experiment Logger는 에르메스 에이전트의 판단 근거와 결과를 저장한다. 본 연구에서 로그는 단순한 부가 정보가 아니라, 프레임워크의 핵심 산출물이다.

로그는 다음 정보를 포함한다.

- 실험 ID
- 기존 설정
- 관찰된 metric
- 에이전트의 상태 해석
- 선택한 행동
- 변경된 설정
- 변경 이유
- 결과 metric
- 다음 행동 여부

## 5. 에르메스 에이전트 실행 규칙

에르메스 에이전트는 ML 실험을 제어하지만, 모든 행동을 자유롭게 수행하지 않는다. 본 연구는 에이전트의 행동을 허용 행동, 금지 행동, 중단 조건으로 구분한다.

### 5.1 허용 행동

에르메스 에이전트가 수행할 수 있는 행동은 다음과 같다.

- learning rate 증가 또는 감소
- batch size 후보군 내 변경
- epoch 수 증가 또는 감소
- optimizer 후보군 선택
- scheduler 적용 여부 결정
- early stopping patience 조정
- checkpoint 기반 재학습 실행
- 새로운 실험 run 생성

### 5.2 금지 행동

에르메스 에이전트가 수행해서는 안 되는 행동은 다음과 같다.

- test set 결과를 기준으로 다음 실험 방향 결정
- 실험 중 validation metric 임의 변경
- 이전 실험 로그 삭제
- 성능이 낮은 실험 결과 은폐
- 데이터 라벨 수정
- 허용 범위를 벗어난 파라미터 변경
- 재현 불가능한 설정으로 실험 실행

### 5.3 판단 규칙 예시

다음은 에르메스 에이전트가 사용할 수 있는 판단 규칙의 예시이다.

```text
IF validation loss가 N epoch 동안 개선되지 않는다
THEN learning rate 감소 또는 early stopping 검토
```

```text
IF train loss는 감소하지만 validation loss가 증가한다
THEN 과적합으로 해석하고 early stopping, regularization, epoch 감소를 검토
```

```text
IF train loss와 validation loss가 모두 높게 유지된다
THEN 과소적합으로 해석하고 epoch 증가, learning rate 조정, optimizer 변경을 검토
```

```text
IF loss curve가 크게 진동한다
THEN learning rate가 과도할 가능성을 검토하고 learning rate 감소 또는 scheduler 적용
```

```text
IF 여러 실험에서 개선 폭이 임계값 이하로 유지된다
THEN 현재 탐색 세션을 중단하고 best run을 선택
```

## 6. 기존 접근과의 차별성

본 연구의 차별성은 에르메스 에이전트가 단순히 하이퍼파라미터를 탐색하는 것이 아니라, 실험 흐름 자체를 해석하고 제어한다는 점에 있다.

| 구분 | 기존 AutoML/HPO | 제안 프레임워크 |
|---|---|---|
| 중심 목표 | 최적 성능 탐색 | 설명 가능한 실험 흐름 제어 |
| 주요 입력 | 탐색 공간, 목적 함수 | 학습 로그, metric 변화, 실험 상태 |
| 주요 출력 | 최적 파라미터 조합 | 다음 실험 결정, 판단 근거, 실험 로그 |
| 판단 방식 | 탐색 알고리즘 중심 | 관찰-해석-결정-기록 중심 |
| 설명 가능성 | 제한적일 수 있음 | 결정 로그를 핵심 산출물로 포함 |
| 사람의 역할 | 탐색 공간 정의자 | 에이전트 실행 원칙 설계자 |

본 연구는 AutoML보다 더 좋은 성능을 항상 보장한다고 주장하지 않는다. 대신 성능, 실험 횟수, 수렴 안정성, 재현성, 결정 로그의 설명 가능성을 함께 평가하는 실험 자동화 구조를 제안한다.

## 9. 논의

### 9.1 연구의 의의

본 연구의 의의는 ML 학습 자동화를 최적 파라미터 탐색에만 한정하지 않고, 실험 의사결정의 구조화와 설명 가능성까지 확장했다는 데 있다. 에르메스 에이전트는 학습 결과를 관찰하고 해석한 뒤, 제한된 권한 안에서 다음 실험 행동을 결정한다. 이를 통해 실험 자동화 과정에서 왜 특정 행동이 선택되었는지를 기록할 수 있다.

### 9.2 장점

제안 프레임워크의 장점은 다음과 같다.

- 반복 실험 과정에서 사람의 수동 조정 부담을 줄일 수 있다.
- 학습 결과에 대한 판단 근거를 로그로 남길 수 있다.
- 실패한 실험도 기록하여 사후 분석이 가능하다.
- test set 사용 제한과 로그 보존 규칙을 통해 실험 신뢰성을 높일 수 있다.
- AutoML과 MLOps 도구 사이의 빈 공간인 실험 의사결정 자동화를 다룰 수 있다.

### 9.3 한계

본 연구는 다음과 같은 한계를 가진다.

- 에이전트의 판단 품질은 실행 규칙과 관찰 지표 설계에 영향을 받는다.
- 복잡한 모델이나 대규모 데이터셋에서는 단순 규칙만으로 충분하지 않을 수 있다.
- 에이전트가 선택한 행동이 항상 최적 성능을 보장하지는 않는다.
- 결정 로그의 설명 가능성은 정량화가 어렵기 때문에 별도의 평가 기준이 필요하다.
- 잘못 설계된 제약은 유용한 실험 가능성을 제한할 수 있다.

### 9.4 확장 가능성

향후 연구에서는 다음 방향으로 확장할 수 있다.

1. 실험 로그를 기반으로 규칙의 효과를 평가하는 반성적 에이전트 구조
2. 여러 실험 전략을 선택하는 메타 에이전트 구조
3. LLM 기반 에이전트를 활용한 실험 로그 요약 및 다음 전략 제안
4. MLflow, Weights & Biases 등 기존 MLOps 도구와의 연동
5. 다중 모델 또는 다중 데이터셋 환경에서의 에이전트 실험 제어

## 10. 결론

본 연구는 머신러닝 학습 과정을 단일 모델 최적화 절차가 아니라, 에르메스 에이전트가 관찰, 해석, 결정, 실행, 기록을 반복하는 실험 제어 과정으로 재정의하였다. 제안 프레임워크에서 ML 모델은 데이터를 학습하고 metric을 생성하며, 에르메스 에이전트는 해당 metric을 기반으로 실험 상태를 해석하고 다음 실험 설정을 결정한다.

본 연구의 핵심은 성능 최적화만이 아니라, 실험 결정 과정의 설명 가능성, 재현성, 기록 가능성을 함께 고려하는 것이다. 따라서 제안 프레임워크는 기존 AutoML이나 하이퍼파라미터 최적화와 경쟁하는 단일 성능 최적화 기법이라기보다, 반복 학습 실험을 더 체계적으로 운영하기 위한 에이전트 기반 실험 제어 구조로 이해할 수 있다.

향후 실제 실험을 통해 에르메스 에이전트가 성능, 실험 횟수, 수렴 안정성, 사람 개입 횟수, 결정 로그 품질 측면에서 어떤 효과를 보이는지 검증할 필요가 있다. 또한 실험 로그를 바탕으로 규칙을 선택하거나 개선하는 메타 에이전트 구조로 확장한다면, 머신러닝 실험 자동화는 단순 반복 실행을 넘어 설명 가능한 실험 지능화로 발전할 수 있다.
